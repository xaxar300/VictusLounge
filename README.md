# VictusLounge Control Center

VictusLounge Control Center - настольное WPF-приложение для управления компьютерным клубом. Проект покрывает основные рабочие сценарии клиента, администратора и владельца: карту ПК, бронирование, игровые сессии, баланс, платежи, смену, тарифы, события и операционную аналитику.

## Назначение

Приложение моделирует рабочее место компьютерного клуба. Через один интерфейс можно:

- видеть загрузку клуба и состояние зон;
- бронировать один или несколько ПК;
- запускать, продлевать и закрывать игровые сессии;
- учитывать платежи и пополнения баланса;
- вести смену администратора;
- отслеживать очередь оплат и сервисные ПК;
- управлять тарифами и смотреть метрики владельца;
- переключать язык, тему и размер интерфейса.

## Основные возможности

### Главная панель

На главной странице показываются ключевые показатели клуба: свободные ПК, активные брони, события, загрузка зон и быстрые действия. Блок "Зоны клуба" оформлен как визуальные карточки зон Standard, VIP, Bootcamp и Royal VIP с количеством мест, свободными ПК, тарифом и кратким описанием.

### Карта клуба

Карта клуба отображает компьютеры по зонам и их текущие состояния. Поддерживаются основные статусы:

- свободен;
- занят;
- забронирован;
- на обслуживании.

При выборе ПК показываются зона, статус, характеристики и доступность бронирования. Статусы ПК синхронизируются с базой и обновляют карту и форму бронирования.

### Бронирование

Форма бронирования поддерживает:

- одиночный режим и бронь для компании;
- выбор даты;
- выбор зоны клуба;
- выбор длительности или пакета;
- выбор времени начала;
- выбор одного или нескольких ПК;
- расчет стоимости, скидки и итоговой суммы.

Итог бронирования закреплен справа: левая часть формы прокручивается отдельно, а сумма, выбранные ПК, дата, время и тариф остаются на виду перед подтверждением.

### Кабинет клиента

Клиент видит персональные данные, активную бронь, активную сессию, баланс, бонусы, прогресс лояльности и историю посещений. Для опасных клиентских действий, таких как отмена брони или завершение сессии, можно включить подтверждение в настройках.

### Баланс и платежи

Раздел баланса показывает текущий баланс, бонусы, пакеты пополнения и историю операций. Поддерживаются промокоды, быстрые игровые пакеты и пополнение через оверлей.

### События

Раздел событий содержит фильтры по типам мероприятий и карточки активностей клуба. Клиент может присоединиться к событию, а интерфейс обновляет статус действия через уведомления.

### Админ-панель

Администратор может:

- создать гостевую сессию;
- отметить оплату;
- закрыть ожидающие оплаты;
- перенести или отменить бронь;
- поставить ПК в сервис;
- снять ПК с сервиса;
- закрыть, оплатить или продлить активную сессию;
- вести журнал операций.

Для обратимых опасных действий добавлен undo snackbar. После перевода ПК в сервис, снятия сервиса или закрытия/повторного открытия смены можно отменить последнее действие через кнопку "Отменить".

### Смена

Раздел смены показывает наличные, онлайн-оплаты, задачи администратора и журнал инцидентов. Смену можно закрыть или снова открыть. Расходы смены сохраняются в финансовом потоке как отрицательные платежи.

### Панель владельца

Владелец видит выручку, загрузку, средний чек, повторные визиты, состояние тарифов и сценарии спроса. Можно переключать режимы спроса, менять тарифы и экспортировать сводку.

## UI и UX

В приложении реализованы:

- три темы: черно-золотая, графитовая и светлая;
- переключение языка RU / EN;
- размер интерфейса: компактный, обычный, крупный;
- сохранение пользовательских настроек между запусками;
- единые status badges для статусов;
- закрепленный итог бронирования;
- глобальный поиск и командная палитра;
- toast-уведомления;
- центр уведомлений;
- undo snackbar для части опасных админских действий.

## Командная палитра и поиск

Верхняя строка поиска работает как быстрый поиск и как командная палитра.

Горячая клавиша:

```text
Ctrl + K
```

Примеры команд:

```text
бронь
карта
админ
смена
настройки
светлая тема
темная тема
графит
компактный
обычный
крупный
ru
en
STD-01
VIP-03
```

Команды навигации открывают соответствующий раздел. Команды темы, языка и размера сразу применяют настройку. Поиск по имени ПК открывает карту клуба и выбирает найденный компьютер.

## Сохранение настроек

Пользовательские настройки сохраняются локально в профиле Windows:

```text
%AppData%\VictusLounge\user-settings.json
```

Сохраняются:

- выбранная тема;
- язык интерфейса;
- размер интерфейса;
- включение подтверждений для опасных клиентских действий.

Если файл настроек отсутствует или поврежден, приложение запускается со стандартными значениями.

## Роли и учетные записи

Тестовые пользователи создаются при инициализации базы.

| Роль | Логин | Пароль |
| --- | --- | --- |
| Owner | `owner` | `owner123` |
| Admin | `admin1` | `admin123` |
| Admin | `admin2` | `admin456` |
| Client | `client1` | `client123` |

Доступ к разделам зависит от роли. Клиент работает с картой, бронью, кабинетом, балансом и событиями. Администратор получает рабочие инструменты смены и админ-панель. Владелец видит управленческие метрики и тарифы.

## Настройка базы данных

Строка подключения по умолчанию находится в `appsettings.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=victus_lounge;Trusted_Connection=True;TrustServerCertificate=True"
  }
}
```

Ее можно переопределить через переменную окружения `VICTUS_DB`.

Примеры для PowerShell:

```powershell
$env:VICTUS_DB="Server=.\SQLEXPRESS;Database=victus_lounge;Trusted_Connection=True;TrustServerCertificate=True"
$env:VICTUS_DB="Server=(localdb)\MSSQLLocalDB;Database=victus_lounge;Trusted_Connection=True;TrustServerCertificate=True"
```

Если база не создана, приложение может создать ее через Entity Framework Core при запуске. Также можно вручную выполнить SQL-скрипт:

```sql
Scripts/CreateDatabase.sql
```

## Технические заметки

- Приложение таргетит `net9.0-windows`, поэтому запускается только на Windows.
- Пароли тестовых пользователей хранятся как SHA-256-хэши с общей солью приложения.
- Внешние ключи используют `Restrict`, чтобы случайно не удалить пользователей или ПК с историей.
- Seed-данные используют фиксированные числовые ID, чтобы связи между пользователями, бронями и сессиями оставались стабильными.
- Журнал админских операций сохраняется как служебные записи в истории платежей.
- `Payment.Amount` используется и для доходов, и для расходов: отрицательная сумма означает расход смены.
- `Shift.EmployeeName` хранится как отображаемый текст, а не как внешний ключ на пользователя, чтобы схема оставалась проще для учебного проекта.
- Локализация обновляет видимые WPF-контролы на текущем visual tree без перезапуска приложения.
- Настройки пользователя сохраняются отдельно от базы данных, поэтому тема, язык и размер интерфейса не зависят от SQL Server.

## Ограничения

- Часть оперативных toast-сообщений остается на русском языке.
- Финансовая модель упрощена для учебного проекта.
- Undo доступен только для обратимых админских операций, где можно безопасно восстановить предыдущее состояние.
- Проект не является production-системой для реального клуба без доработки прав доступа, аудита, кассовой интеграции и резервного копирования.
